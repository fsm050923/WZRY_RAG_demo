本项目是一个专注于《王者荣耀》装备领域的问答系统。用户可以使用自然语言提问，系统根据知识库中的装备文档，检索最相关的信息，并利用大语言模型生成准确答案。项目核心为 RAG（检索增强生成）架构，并通过多种优化手段提升检索精度与答案质量。

## 🧰 技术栈

- **语言与框架**：Python 3.10+, Streamlit
- **RAG 框架**：LangChain
- **嵌入模型**：BAAI/bge-m3（多语言高精度嵌入）
- **重排序模型**：BAAI/bge-reranker-v2-m3（交叉编码器）

## ⚙️ 优化点详解

### 1. HyDE（假设文档嵌入）
- **问题**：用户提问可能过于简短，与文档关键词不匹配。
- **解决**：先用大模型生成一段假设性答案（即使不准确），然后用它检索，有效提升召回。

### 2. 混合检索（BM25 + 向量）
- **BM25**：关键词精确匹配，保证专有名词（如装备名）不丢失。
- **向量检索**：语义相似度搜索，捕捉相关但表述不同的查询。
- **权重**：`[0.3, 0.7]`，兼顾精确与泛化。

### 3. 重排序（Reranking）
- **原理**：交叉编码器同时输入问题和文档，计算精确相关性分数。
- **优势**：比双编码器（向量检索）更准，弥补初步检索的排序缺陷。

### 4. 相关性阈值
- **作用**：当最相关文档的分数低于 `THRESHOLD=0.5` 时，认为无相关答案，返回空列表。
- **效果**：避免在用户寒暄（如“你好”）时错误返回文档，提升对话体验。

### 5. 模块化设计
- 配置、工具、检索器、界面分离，便于后期扩展（如增加缓存、日志、更多检索策略）。
